# Remember Skill

整理和提升关键信息到长期记忆。

## 记忆层次

| 层次 | 存储位置 | 内容 |
|------|---------|------|
| **CLAUDE.md** | 项目根目录 | 所有参与者应遵循的项目规范 |
| **CLAUDE.local.md** | 项目根目录 | 仅适用于当前用户的个人偏好 |
| **团队记忆** | 共享位置 | 跨项目适用的组织级知识 |
| **自动记忆** | 会话内存 | 工作笔记、临时上下文 |

## 工作流程

### 1. 收集所有记忆层

读取现有的记忆文件：
- CLAUDE.md（项目规范）
- CLAUDE.local.md（个人偏好）

### 2. 分类每个记忆条目

将记忆分配到最佳位置：

**CLAUDE.md**：
- 项目编码规范（如"使用 bun 而非 npm"）
- API 路由命名规范（如"使用 kebab-case"）
- 测试命令（如"bun test"）
- 偏好函数式风格

**CLAUDE.local.md**：
- 用户响应偏好（如"简洁回复"）
- 解释权衡决策
- 不要自动提交
- 提交前运行测试

**团队记忆**：
- 部署流程（如"PR 通过 #deploy-queue"）
- 暂存环境位置
- 基础设施归属

**保留在自动记忆**：
- 工作笔记
- 临时上下文
- 不确定模式

### 3. 识别清理机会

- **重复**：已在新位置存在的内容 → 提议删除
- **过时**：被新信息矛盾的旧条目 → 提议更新
- **冲突**：互相矛盾的内容 → 提议解决方案

### 4. 输出报告

```markdown
# 记忆整理报告

## 提升项（Promotions）
- [条目] → [目标位置]
- 原因

## 清理项（Cleanup）
- 重复/过时/冲突的条目

## 模糊项（Ambiguous）
- 需要用户确认的条目

## 无需操作项
- 保持不变的原因
```

## 使用示例

```
用户: 帮我整理一下记忆
Agent:
1. 读取现有的 CLAUDE.md 和 CLAUDE.local.md
2. 分析当前记忆条目
3. 识别整理机会
4. 呈现整理建议
5. 用户批准后执行变更
```

## 注意事项

- **先呈现，再修改** - 始终先展示建议，获得批准后再变更
- **不明确时询问** - 不要猜测，询问用户
- **保持简洁** - CLAUDE.md 只包含对 Claude 的指示，不包含用户外部工具偏好
