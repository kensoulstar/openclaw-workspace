# Debug Skill

帮助诊断和解决代码问题。

## 工作流程

### 1. 理解问题

首先明确：
- **问题现象**：程序做了什么？应该做什么？
- **发生场景**：在什么操作/输入时出现问题？
- **错误信息**：完整的错误消息或日志

### 2. 收集信息

收集调试所需的信息：

```
## 问题上下文
- 相关代码文件
- 最近一次变更
- 环境配置
- 错误堆栈（如有）
```

### 3. 分析根因

使用以下方法定位问题：

| 方法 | 适用场景 |
|------|----------|
| **日志分析** | 运行时错误、状态异常 |
| **断点调试** | 逻辑错误、变量值异常 |
| **代码审查** | 静态分析、逻辑错误 |
| **重现代码** | 最小化复现问题 |

### 4. 制定修复方案

分析完成后，提供：

```
## 问题诊断

### 根本原因
[解释问题发生的根本原因]

### 影响范围
[这个问题影响了哪些功能]

## 修复方案

### 推荐方案
[最直接的修复方式]

### 替代方案
[其他可能的解决方式]

### 预防措施
[如何避免类似问题]
```

## 常见问题类型

### 1. 运行时错误

```
诊断步骤：
1. 查看错误堆栈信息
2. 定位错误发生位置
3. 分析错误原因
4. 检查相关变量值
5. 验证修复
```

### 2. 逻辑错误

```
诊断步骤：
1. 理解预期行为
2. 追踪实际行为
3. 找出分歧点
4. 分析代码逻辑
5. 修正逻辑
```

### 3. 性能问题

```
诊断步骤：
1. 识别瓶颈位置
2. 分析时间复杂度
3. 检查资源使用
4. 优化算法或实现
```

### 4. 配置问题

```
诊断步骤：
1. 验证配置文件
2. 检查环境变量
3. 确认依赖版本
4. 核对文档要求
```

## 使用示例

```
用户: 这个函数返回的结果不对
Agent:
1. 询问具体的输入和预期输出
2. 分析函数逻辑
3. 找出返回值计算的问题
4. 提供修复方案
```

## 注意事项

- 先理解问题再开始调试
- 记录每一步的发现
- 验证修复后要进行回归测试
- 如果无法确定原因，提供最可能的假设
