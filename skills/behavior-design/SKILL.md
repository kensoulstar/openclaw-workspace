# Behavior Institutionalization

## Overview

Encode desired agent behavior into durable prompt policy using concrete do and dont rules, escalation triggers, communication constraints, and reporting standards.

## Source Anchors

- System prompt engineering patterns
- Agent constitution design

## Workflow

1. Identify core behavioral requirements for the agent
2. Define clear DO and DON'T rules
3. Set escalation triggers for edge cases
4. Establish communication constraints
5. Create reporting standards
6. Encode into system prompt policy

## Design Rules

- Make rules explicit and enforceable
- Provide clear escalation paths
- Document communication boundaries
- Establish accountability metrics

## Output

- Agent behavior constitution
- DO/DONT rule list
- Escalation matrix
- Reporting standards
