# Stuck Skill

当 Agent 陷入困境时提供帮助。

## 问题类型识别

### 1. 死循环
**特征**：
- CPU 持续 90%+
- 相同输出重复出现
- 超时无响应

**诊断方法**：
```
1. 检查 Agent 日志
2. 分析最近的代码变更
3. 识别可能的无限循环模式
```

### 2. 等待资源
**特征**：
- 进程状态 D（不可中断睡眠）
- 等待 I/O 或网络响应

**诊断方法**：
```
1. 检查网络连接
2. 检查外部 API 状态
3. 检查文件/数据库锁定
```

### 3. 逻辑卡住
**特征**：
- 反复尝试相同方法
- 无法找到解决方案
- 缺少必要信息

**诊断方法**：
```
1. 重新理解问题
2. 寻找其他解决路径
3. 询问用户补充信息
```

## 解决策略

### 策略 1: 重启思路
- 回顾原始需求
- 重新分析问题
- 尝试不同方法

### 策略 2: 信息补充
- 询问用户缺失信息
- 提供选项让用户选择
- 缩小问题范围

### 策略 3: 分步验证
- 将复杂问题分解
- 逐一验证每个部分
- 定位问题所在

## 使用示例

```
用户: Agent卡住了，一直在重复同样的操作
Agent:
1. 分析最近的操作日志
2. 识别重复模式
3. 建议终止当前操作
4. 提供新的解决思路
```
