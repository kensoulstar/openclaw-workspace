# Blast Radius Permission

## Overview

Design or audit permission systems that combine rule-based allow ask deny logic, safety checks, auto-mode classifiers, dangerous-rule stripping, and blast-radius-aware user confirmation.

## Source Anchors

- Permission system architecture
- Risk assessment frameworks

## Workflow

1. Define permission boundaries
2. Implement allow/ask/deny logic
3. Add safety checks
4. Configure blast-radius awareness
5. Set up user confirmation flows

## Design Rules

- Principle of least privilege
- Blast radius containment
- Graduated permission levels
- Audit trail for all decisions

## Output

- Permission matrix
- Risk assessment framework
- Confirmation workflows
