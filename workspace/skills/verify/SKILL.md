# Verify Skill

验证代码变更是否按预期工作。

## 工作流程

### 1. 识别变更

首先了解用户要求验证的内容：

- 如果用户指定了具体功能或变更，聚焦验证该部分
- 如果用户没有明确说明，询问他们想要验证什么

### 2. 收集上下文

在执行验证前，收集必要信息：

```
- 变更涉及的文件
- 预期行为是什么
- 如何运行/测试这个功能
- 可能的边界情况
```

### 3. 验证执行

根据变更类型选择验证方式：

| 变更类型 | 验证方式 |
|----------|----------|
| **Bug修复** | 复现原bug场景，验证修复 |
| **新功能** | 测试功能正常，边界情况 |
| **配置变更** | 检查配置生效 |
| **代码重构** | 验证功能不受影响 |

### 4. 报告结果

验证完成后，报告：

```
## 验证结果

### ✅ 通过项
- [具体测试项和结果]

### ❌ 失败项
- [具体问题描述]
- 原因分析
- 建议修复

### ⚠️ 需要注意
- [潜在风险或边界情况]
```

## 使用示例

```
用户: 验证我修复的登录bug
Agent: 
1. 了解修复内容
2. 测试登录流程
3. 检查边界情况（错误密码、空用户等）
4. 报告验证结果
```

## 注意事项

- 如果无法直接运行，尝试通过代码审查验证逻辑正确性
- 对于复杂功能，分步骤验证每个子功能
- 记录验证过程中的任何发现
